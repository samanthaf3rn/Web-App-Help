Fibonacci

 fibonacci series
function fibb() {
  var n = parseInt(document.form1.n.value);
  var f1 = 0, f2 = 1, f3, i = 0;
  el.innerHTML = f1 + ' ' + f2 + ' ';
  while (i < n - 2) {
    f3 = f1 + f2;
    el.innerHTML += f3 + ' ';
    f1 = f2; f2 = f3; i++;
  }
}

Storage & AJAX

localStorage, sessionStorage, Cookies, and XMLHttpRequest.

 localStorage persistent
// Save
localStorage.setItem(key, message);

// Read
var val = localStorage.getItem(key);

// Remove one key
localStorage.removeItem(key);

// Clear everything
localStorage.clear();

// Note: survives browser/tab close
 sessionStorage tab only
// Same API as localStorage
sessionStorage.setItem(key, message);
var val = sessionStorage.getItem(key);

// Cleared when tab is closed
// Use for temp session data (like cart)
 Cookies (JS)
// Set cookie — 3 day expiry
var date = new Date();
date.setTime(date.getTime() + (3*24*60*60*1000));
document.cookie = "username=" + user
  + "; expires=" + date.toUTCString()
  + "; path=/";

// Read all cookies
var all = document.cookie.split(';');
for (var i = 0; i < all.length; i++) {
  display += all[i].trim() + '<br>';
}
 AJAX — XMLHttpRequest
function loadData() {
  var xhr = new XMLHttpRequest();
  xhr.open('GET', 'data.txt', true);
  xhr.send();

  xhr.onload = function() {
    if (xhr.status === 200) {
      document.getElementById('data')
        .innerHTML = xhr.responseText;
    } else {
      document.getElementById('data')
        .innerHTML = 'Failed to load';
    }
  };
  xhr.onerror = function() {
    document.getElementById('data')
      .innerHTML = 'Error in request';
  };
}
<!-- trigger on keypress -->
<input onkeypress="loadData()">
ℹ Comparison: localStorage = permanent; sessionStorage = tab-scoped; Cookies = can be sent to server with every request (PHP reads via $_COOKIE).

Controller — Full CRUD

 StudentController.php
class StudentController extends Controller {

  // ADD
  function add(Request $request) {
    $s = new Student();
    $s->name  = $request->name;
    $s->email = $request->email;
    $s->phone = $request->phone;
    $s->save();
  }

  // LIST
  function list() {
    $data = Student::all();
    return view('list-student', ['student123' => $data]);
  }

  // DELETE
  function deleteStudent($id) {
    Student::where('id', $id)->delete();
    return redirect('listStudent');
  }

  // UPDATE — load form
  function updateStudent($id) {
    $s = Student::find($id);
    return view('updateform', ['studentdata' => $s]);
  }

  // UPDATE — save
  function updateform($id, Request $req) {
    $s = Student::find($id);
    $s->name = $req->name;
    $s->save();
    return redirect('listStudent');
  }
}

Viva Questions

Theory-based questions & answers from every topic — exam ready.

HTML

Q What is HTML? What does DOCTYPE do?
HTML (HyperText Markup Language) is the standard language for creating web pages. It uses tags (elements) to structure content.

<!DOCTYPE html> tells the browser which version of HTML is being used. In HTML5 it is simply <!DOCTYPE html> — without it, the browser enters "quirks mode" and may render the page incorrectly.
Q Difference between GET and POST methods?
GET: Data sent in the URL query string. Visible to user. Bookmarkable. Limited data size. Used for retrieving data.
  → ?Name=John&Email=john@x.com

POST: Data sent in the HTTP body. Not visible in URL. No size limit. Used for sending/saving data (forms, logins, file uploads).
Q What is colspan and rowspan?
colspan: Merges a cell across multiple columns horizontally.
  → <th colspan="3"> spans 3 columns wide.

rowspan: Merges a cell across multiple rows vertically.
  → <td rowspan="2"> spans 2 rows tall.

Both are used to create complex table layouts like timetables.
Q What are meta tags? Name a few.
Meta tags provide metadata about the HTML document. They go inside <head> and are not displayed.

charset  — character encoding (UTF-8)
viewport — responsive scaling on mobile
description — page summary for SEO
keywords — search engine keywords
author   — page author
Q What is the hidden input type used for?
type="hidden" sends data with a form that the user cannot see or change in the browser UI.

Common uses: passing IDs, CSRF tokens, record identifiers between pages.

Example from journal:
<input type="hidden" name="secret" value="iambatman">
Q Difference between <th> and <td>?
<th> — Table Header cell. Displayed bold and centered by default. Used for column/row headings.

<td> — Table Data cell. Normal weight. Used for actual data content.

Both sit inside <tr> (table row).

CSS

Q What is CSS? What are the 3 ways to apply CSS?
CSS (Cascading Style Sheets) controls the visual presentation of HTML elements.

1. Inline  — style attribute on the element itself
2. Internal — <style> block inside <head>
3. External — separate .css file linked via <link>

External CSS is preferred (reusable, maintainable).
Q What is a CSS selector? Name types.
A selector targets which HTML elements a style rule applies to.

Element  → h1 { }        targets all <h1>
Class    → .box { }     targets class="box"
ID       → #header { }  targets id="header"
Pseudo   → a:hover { }  targets on hover
Universal→ * { }        targets everything
Q What is Flexbox? Key properties?
Flexbox is a CSS layout model for arranging items in a row or column with flexible sizing.

display: flex         — enables flex container
flex-direction        — row / column
justify-content       — align on main axis (center, space-between…)
align-items           — align on cross axis
flex-wrap             — allow wrapping to next line
Q What does @media print do?
@media print is a CSS media query that applies styles only when the page is printed.

Used in the certificate page to hide the print button:
@media print { .print-btn { display: none; } }

This keeps the button visible on screen but removes it from the printed output.

JavaScript

Q What is JavaScript? Where can scripts be placed?
JavaScript is a client-side scripting language that makes web pages interactive — it runs in the browser.

Internal  — <script> inside <head> or <body>
External  — separate .js file via <script src="...">
Inline    — directly in event attributes (onclick="...")

Best practice: external file loaded before </body>.
Q What are the 3 types of functions in JS?
1. Named function
   function add(a,b) { return a+b; }
   Hoisted — can be called before definition.

2. Anonymous function
   let add = function(a,b) { return a+b; };
   Assigned to a variable, not hoisted.

3. Arrow function
   let add = (a,b) => a+b;
   Concise syntax, no own 'this' binding.
Q Difference between innerHTML and innerText?
innerHTML — Gets/sets content including HTML tags.
  el.innerHTML = "<h2>Hello</h2>"; // renders as heading

innerText — Gets/sets content as plain text only.
  el.innerText = "<h2>Hello</h2>"; // shows literally as text

Use innerHTML when you want to inject HTML elements.
Use innerText for safe plain text (avoids XSS).
Q What are DOM events? Name mouse & key events.
DOM Events are actions the browser detects that you can respond to with JS.

Mouse events:   onclick, ondblclick, onmousedown,
                onmouseup, onmouseover, onmouseout

Key events:     onkeydown  — any key pressed
                onkeypress — character key pressed
                onkeyup    — key released

addEventListener is preferred over inline handlers.
Q What does getElementById return? What if not found?
getElementById(id) returns a single Element object matching the given id.

If no element is found → returns null.

Other selectors:
getElementsByClassName → HTMLCollection (live)
getElementsByTagName   → HTMLCollection (live)
querySelector         → first match (static)
querySelectorAll      → NodeList (all matches)
Q What is isNaN()? What does it return?
isNaN() — "is Not a Number" — checks if a value cannot be converted to a number.

isNaN("hello")  → true   (not a number)
isNaN("123")    → false  (can be converted)
isNaN(42)       → false

Storage & AJAX

Q Difference between localStorage and sessionStorage?
localStorage
  • Data persists even after browser/tab is closed.
  • Scope: same origin (protocol + domain + port).
  • Use for: user preferences, themes, saved data.

sessionStorage
  • Data is cleared when the tab or browser closes.
  • Scope: same tab only.
  • Use for: temporary cart, one-session form data.

Both use the same API: setItem, getItem, removeItem, clear.
Q What are cookies? How are they different from localStorage?
Cookies are small pieces of data stored in the browser and sent to the server with every HTTP request.

Differences vs localStorage:
Server access — cookies: yes. localStorage: no.
Expiry        — cookies: set manually. localStorage: permanent.
Size limit    — cookies: ~4KB. localStorage: ~5MB.
PHP reads     — via $_COOKIE['name'].
Q What is AJAX? What is it used for?
AJAX — Asynchronous JavaScript And XML.

It allows web pages to send and receive data from a server without reloading the page.

How it works:
1. JS creates an XMLHttpRequest object.
2. Sends a request to the server (GET/POST).
3. Server responds (text, JSON, XML).
4. JS updates part of the DOM with the result.

Used in live search, infinite scroll, form submissions.
Q What does xhr.status === 200 mean?
HTTP status codes indicate the result of a request:

200 OK           — request succeeded, response is ready
404 Not Found    — resource doesn't exist
500 Server Error — something broke on the server
403 Forbidden   — no permission to access

In AJAX: xhr.status === 200 confirms a successful response before using xhr.responseText.

Using Bootstrap via CDN means:

PHP

Q What is PHP? How is it different from JavaScript?
PHP — Hypertext Preprocessor. A server-side scripting language used to build dynamic web pages and interact with databases.

PHP runs on the SERVER — output is HTML sent to browser.
JavaScript runs in the BROWSER (client-side).

PHP can: connect to MySQL, handle forms, manage sessions, read/write files. JS cannot do these directly.
Q What is the difference between echo and print in PHP?
echo  — outputs one or more strings. No return value. Slightly faster. Can take multiple comma-separated arguments.
  echo "Hello", " World";

print — outputs one string. Always returns 1 (so it can be used in expressions). Takes only one argument.
  $x = print("Hello"); // $x = 1

Both are language constructs, not functions — no parentheses needed.
Q What is a PHP session? How does it work?
A session stores user data on the server side between page requests. Unlike cookies, session data is not stored in the browser.

How it works:
1. session_start() — creates or resumes a session.
2. PHP assigns a unique session ID (stored in a cookie).
3. Data stored in $_SESSION['key'] on the server.
4. session_destroy() ends the session (logout).
Q What is mysqli? What is a prepared statement?
mysqli — "MySQL Improved" — PHP extension for connecting to MySQL databases. Supports both procedural and object-oriented style.

Prepared statement: A pre-compiled SQL query where user input is bound as parameters (not injected directly into SQL). Protects against SQL injection.

$stmt = $conn->prepare("SELECT * FROM users WHERE id=?");
$stmt->bind_param("i", $id);
$stmt->execute();
Q What is define() vs a variable in PHP?
Variable: prefixed with $. Value can be changed any time.
  $greeting = "Hello";

define(): creates a constant. No $ sign. Value cannot be changed after definition.
  define('GREETING', 'Hello');
  echo GREETING;

Constants are global by default (accessible anywhere in the script).
Q What is the difference between $_GET, $_POST, $_REQUEST?
$_GET     — collects data sent in the URL query string (form method="GET").
$_POST    — collects data sent in the HTTP body (form method="POST"). More secure.
$_REQUEST — collects data from both GET and POST (and cookies). Convenient but less specific.

Best practice: use $_POST for sensitive data (passwords), $_GET for search/filter queries.

XML & JSON

Q What is XML? What are its rules?
XML — eXtensible Markup Language. A text-based format for storing and transporting structured data. Tags are user-defined (unlike HTML).

Rules (well-formed XML):
• Must have a single root element
• All tags must be closed
• Tags are case-sensitive
• Attributes must be quoted
• Proper nesting required
Q What is JSON? How does it differ from XML?
JSON — JavaScript Object Notation. Lightweight data format using key-value pairs.

JSON:  {"name":"Ana","age":20}
XML:   <person><name>Ana</name></person>

JSON is:  Smaller, faster to parse, native to JS.
XML is:   More verbose, supports attributes, better for docs.

JSON.parse()     — text string → JS object
JSON.stringify() — JS object → text string

Laravel MVC

Q What is MVC architecture?
MVC — Model, View, Controller. A software design pattern that separates an application into 3 components:

Model      — handles data & database logic (Eloquent ORM in Laravel)
View       — handles the UI/presentation layer (Blade templates)
Controller — handles user input, links Model and View

Benefit: separation of concerns — easier to maintain, test, and scale.
Q What is Laravel? What is Artisan?
Laravel is a PHP MVC web framework with an elegant syntax. It provides routing, Eloquent ORM, Blade templating, migrations, and more out of the box.

Artisan is Laravel's command-line interface (CLI). It generates boilerplate code and runs tasks:

php artisan serve            — start dev server
php artisan make:model X -mcr — model+migration+controller
php artisan migrate           — run migrations
Q What is a Migration in Laravel?
A migration is a PHP file that defines the structure of a database table in code — like version control for your database schema.

up()   — creates/modifies the table (runs on migrate)
down() — reverses the change (runs on rollback)

Benefits: team members get the same DB structure by running php artisan migrate — no manual SQL needed.
Q What is @csrf in a Blade form? Why is it needed?
@csrf generates a hidden input field containing a CSRF token (Cross-Site Request Forgery protection).

Without it, Laravel rejects POST requests with a 419 error.

Why: CSRF attacks trick a user's browser into making unauthorized requests to a site they're logged into. The token proves the request came from your own form.
Q What is Eloquent ORM? What does Student::all() do?
Eloquent is Laravel's built-in Object Relational Mapper (ORM). It lets you interact with database tables using PHP objects instead of writing raw SQL.

Each Model = one database table.

Student::all()         — SELECT * FROM students
Student::find($id)     — SELECT * WHERE id = ?
Student::where(...)    — custom WHERE clause
$s->save()            — INSERT or UPDATE
Student::where()->delete() — DELETE
Q What is Blade templating? Name 3 directives.
Blade is Laravel's templating engine. Files use .blade.php extension and allow PHP logic with clean syntax.

{{ $var }}          — Echo and escape output (XSS safe)
{!! $var !!}       — Echo raw unescaped HTML
@foreach(...)      — Loop through arrays/collections
@if / @else / @endif — Conditionals
@csrf              — CSRF token field
@extends / @section — Template inheritance

Web Hosting & General

Web hosting is a service that provides server space for your website files to be accessible on the internet.

Q What is an SSL certificate? What is HTTPS?
SSL — Secure Sockets Layer (now TLS). A digital certificate that encrypts data between the browser and server.

HTTP  — data sent as plain text (insecure)
HTTPS — data encrypted via SSL/TLS (secure, padlock icon)

Benefits: protects passwords, prevents eavesdropping, required for payment pages, improves SEO ranking.

Free SSL available via Let's Encrypt.
Q What is the difference between client-side and server-side?
Client-side: Code runs in the user's browser.
  Technologies: HTML, CSS, JavaScript
  Can: manipulate DOM, validate forms, animate UI.
  Cannot: access database, read server files.

Server-side: Code runs on the web server.
  Technologies: PHP, Node.js, Python
  Can: query DB, handle auth, process payments.
  User never sees the source code.
Q What is SSI (Server Side Includes)?
SSI — Server Side Includes. A simple server-side scripting language that embeds dynamic content into HTML pages.

In PHP context (from your journal), SSI refers to including/requiring other PHP files:
  include("header.php")  — includes file, warns if missing
  require("dbcon.php")   — includes file, dies if missing

Used to reuse database connections, headers, footers across pages.
